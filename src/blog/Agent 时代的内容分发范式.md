上周我在用 Claude Code 的时候，突然意识到一件事。

我给它装了几个 skill，它就能按照我的习惯帮我做事——记住我的偏好，理解我的上下文，每次交互都比上一次更精准。这不就是推荐算法在做的事吗？

只不过，这个"推荐算法"不属于任何平台。它属于我。

## 字节跳动赢在哪里

回头看移动互联网这十年，字节跳动做对了一件事：**用推荐算法重新定义了内容分发**。

在它之前，你想看内容，要么去门户网站看编辑选的，要么去搜索引擎自己找。字节跳动说：你不用选，我比你更懂你想看什么。今日头条、抖音、TikTok，全是这套逻辑。

这套逻辑的核心是一个飞轮：更多用户 → 更多行为数据 → 更好的推荐 → 用户更离不开。数据飞轮转得越快，护城河越深。后来者想追，得先攒够同等量级的数据——这几乎不可能。

所以过去十年，**个性化能力是平台的核心资产**。你作为用户，只是这个飞轮上的一个数据点。

## 一个正在发生的变化

但现在，情况不同了。

AI agent 是你的私人助手。它和你对话，记住你说过的每一句话，理解你的偏好、预算、顾虑。它不需要猜你"可能喜欢什么"——你直接告诉它。

想想这意味着什么。

推荐算法需要你刷几百条内容，通过你的点击、停留、划走来推断你的口味。这是**隐性反馈**，效率很低，而且经常猜错。

Agent 只需要你说一句"太贵了"或者"我不喜欢这个品牌"。这是**显性反馈**，一句话就完成了传统推荐系统几十次点击才能推断的事。

反馈门槛从"行为"降到了"说话"。这是质变，不是量变。

## 从 Feed 到 RFP

这个变化深层的含义是什么？

不是"AI 让推荐算法更好了"。所有人都在说这个，所有人都错了。

真正发生的事是：**推荐这个动作本身，换了主体。**

过去是平台推给你——你打开 App，刷 Feed，算法决定你看什么。

未来是你的 agent 替你去找——你说出需求，agent 向多个内容源发出请求，拉回来给你筛选和组合。

用一个不太恰当但很锐利的比喻：**Feed 变成了 RFP**。

Feed 是平台的独白：我觉得你想看这个。

RFP（Request for Proposal，采购意向书）是用户的主张：这是我的需求，谁能满足？

主语变了。一切都变了。

## 拆开来看：四层变化

把"信息到达用户"这条链路拆成四层，变化就很清晰了。

**供给层。** 过去，内容和商品必须入驻平台才能被用户发现——经销商得在汽车之家开通会员，房东得在贝壳挂房源。未来，供给方可以通过 API、MCP server 等方式直接对接用户的 agent。平台的"独占"被打破。供给方的竞争从"争平台流量"变成"争 agent 信任"——谁的数据更准确、更结构化，谁就更容易被 agent 采纳。

**分发层。** 过去，推荐算法是分发的唯一引擎。未来，用户的 agent 就是分发引擎。平台从"分发方"降级为"供给端点"——它还有内容，但不再垄断用户注意力的分配权。

**个性化层。** 过去，个性化基于隐性信号，数据飞轮在平台侧。未来，个性化基于显性对话，数据飞轮在用户侧。而且这个飞轮是跨场景的——你买车时积累的偏好（预算敏感度、品牌偏好、决策风格），在你下次买保险或选学校时还能用。飞轮的生命周期不是"一次消费"，而是"用户的一生"。

**交互层。** 过去，你是消费者——浏览、点击、消费、下一条。未来，你是委托人——告诉 agent 你要什么，它去获取、筛选、组合、呈现。信息过载消失了，因为 agent 的激励是帮你高效决策，不是让你多刷。

## 一个具体的例子

拿我熟悉的汽车行业来说。

**现在的体验：** 你想买一辆新能源 SUV。打开汽车之家，首页推了一堆车型，有些是广告，有些是算法猜你可能感兴趣的。你点进去看参数，看评测，看口碑。然后打开懂车帝再看一遍。再去小红书搜真实车主的吐槽。花了三天，信息散落在五六个 App 里，自己在脑子里做对比。

**Agent 时代的体验：** 你告诉 agent："帮我看看 25 万以内、适合家庭、续航 500km 以上的 SUV，我比较在意后排空间和辅助驾驶。"Agent 从多个数据源拉取信息——车型参数、专业评测、车主口碑、经销商实时报价——组合成一份定制化的对比报告。你看完说"太贵了，再看看 20 万以内的"或者"比亚迪我不考虑"。Agent 立即调整，下一轮推荐精准了一个量级。

三轮对话，相当于你在汽车之家刷了三天。

## 什么会先变，什么不会

不是所有场景都会被颠覆。两个变量决定了顺序。

**第一个变量：决策成本。** 买车、买房、选医院、选学校——这些高决策成本的场景会最先被改变。因为用户有动力和 agent 深度交互，因为信息维度太多传统算法处理不了（推荐算法很难从你的点击行为推断出"你怕噪音"或者"你家有两个孩子需要七座"），更因为当前体验太差了。

刷短视频、看新闻——这些低决策成本的场景变化最慢。用户追求的是"被投喂"的快感，甚至享受"被意外推荐"的惊喜。推荐算法在这里仍然是王者。

**第二个变量：数据可获取性。** Agent 能不能拿到数据？商品参数、价格、库存这些结构化数据相对容易获取。但平台独家内容、需要登录才能访问的 UGC、经销商的底价——这些被平台锁着。

两个变量画出四个象限：

- **理想区**（高决策 + 数据可获取）：最先被颠覆。有开放 API 的垂直媒体平台首当其冲。
- **攻坚区**（高决策 + 数据封锁）：需要突破平台壁垒。买车买房目前在这里。
- **先驱区**（低决策 + 数据可获取）：已经有雏形了，比如 Perplexity 的购物功能。
- **观望区**（低决策 + 数据封锁）：抖音的地盘，暂时动不了。

## 这不是第一次了

回顾互联网的历史，分发范式每十年换一次主体。

Web 1.0 时代，编辑决定你看什么。新浪首页、搜狐首页，编辑是分发的主体。

Web 2.0 时代，算法决定你看什么。字节跳动的推荐、Google 的排序，算法是分发的主体。

现在，Agent 时代，**你自己决定你看什么**——通过你的 agent，用你自己的语言。

每一次转变都不是"上一代更好了"，而是"换了一个主体"。编辑没有变成更好的编辑，是算法替代了编辑。算法也不会变成更好的算法——是用户的 agent 替代了算法。

## 平台会怎么反应

这是最值得观察的部分。

当平台意识到 agent 正在绕过它的推荐层，它有三条路：

**封锁。** 加强反爬，关闭 API，把数据锁得更死。短期有效，但长期来看，封锁数据的平台会失去 agent 生态的流量入口——就像当年拒绝被搜索引擎收录的网站，最终被遗忘。

**拥抱。** 主动提供 MCP server 或结构化 API，让 agent 方便地获取数据。平台从"分发方"转型为"供给方"——虽然失去了推荐层的溢价，但保住了数据源的价值。

**融合。** 自己做 agent，在平台内部提供 agent 交互体验。这是最可能的路径，但也最危险——因为用户的 agent 是跨平台的，而平台的 agent 只能看到自己的数据。

我的判断是，大多数平台会先封锁，然后在流量下降的压力下被迫拥抱。就像报纸先反抗互联网，最后不得不上线一样。

## 还有一个没解决的问题

信任。

推荐算法时代，我们已经习惯了竞价排名——平台收了钱，把某些内容推到你面前。你知道这件事，但大多数时候无能为力。

Agent 时代，这个问题不会消失，只是换了形态。Agent 的推荐会不会被供给方"收买"？用户怎么知道 agent 推荐某辆车，是因为它真的适合你，还是因为经销商付了推广费？

这是一个开放问题。但有一点是确定的：**当用户可以查看、编辑、甚至导出自己的偏好图谱时，透明度天然就比黑箱算法高。** 你看不懂字节跳动的推荐算法，但你可以看懂自己告诉 agent 的每一句话。

## 写在最后

我在汽车行业做了几年产品，一直在和推荐算法打交道——优化曝光到留资的转化，设计 A/B 实验测试不同的分发策略。我很清楚推荐算法的威力，也很清楚它的局限。

它最大的局限不是技术——技术一直在进步。它最大的局限是**立场**。

平台的推荐算法永远在优化平台的目标函数：DAU、时长、广告收入。不是用户的目标函数。

Agent 不同。它的目标函数是你的目标函数。你说"帮我找一辆合适的车"，它的全部激励就是帮你找到那辆车——不是让你多刷几篇车评，不是让你多看几条广告。

> 当个性化能力从平台手里交还给用户，互联网的底层权力结构就变了。这不是一次算法升级，这是一次权力转移。

我们正好站在这个转折点上。
