Dan Shipper 做了一件很有意思的事。

他本来想解决的问题是溯源——一份文档里，哪句话是人写的，哪句是 AI 写的，谁对哪段内容负责。他觉得这是一个真实的痛点，于是花了几个月做 Mac 应用，实现了左侧色条，紫色是 Agent 写的，绿色是人类写的，还加了 sub-agent 集成和一堆功能。

然后他发现，没人用溯源。所有人疯狂用的，是另一个几乎没花什么功夫做的功能：分享。

把文档链接丢给同事，对方直接进来就能看、就能写，不需要账号，不需要安装任何东西——仅此而已。但这一件事，让 Every 全团队都开始每天用它做计划、改代码、写日记、做头脑风暴。

这是 Proof，Every 刚开源的一个实时协作 Markdown 编辑器。

---

这个"以为用户要 A，结果用的是 B"的故事，如果只当成一个产品教训来读，就太浅了。

更值得想的是：**为什么"分享"这么重要？** 以前 Google Docs 也能分享，Notion 也能分享，为什么 Proof 的分享会让人上瘾？

区别在于：Proof 的分享对象不只是人，还有 Agent。

你把一个链接丢给你的 Claude，它不需要登录，不需要你给它安装插件，它直接就能进去读文档、写内容、留评论。这个链接是它进入你工作流的入口，低摩擦到几乎感觉不到它的存在。

一旦这件事变得顺滑，你和 Agent 合作的方式就变了。你不再是在对话框里来回发消息，而是在一个共同的「地方」工作——你进去改一段，它进去补一段，你们的内容交织在同一个文档里，绿色和紫色，清晰可见。

Dan 说了一个词：**AX，Agent Experience**，和 UX 并列的概念。如果你想优化 Agent 的使用体验，最快的方法就是直接问它——它不会说"我改天再试"，它会立刻告诉你它在哪里卡住了。这让 AX 的迭代速度比 UX 快了一个数量级，也让"为 Agent 设计"这件事，第一次变得和"为用户设计"一样具体、一样可操作。

---

但 Agent-First 产品真正难的问题，不是"Agent 怎么进来"，而是"多个 Agent 同时进来会发生什么"。

Dan 在节目里直接说，他们试过让所有 Claude 同时更新一个文档，结果文档"疯了"。内容被反复重写，段落三倍膨胀，没人能用。

他给出的解释很精准：人类在协作文档里是有社交成本约束的——你不会随便删别人写的东西，因为你要承担后果。**但 Agent 没有这个约束。它有能力，没顾虑，所以它就真的会把别人写的东西删掉重写。**

这不只是个技术问题，是个设计假设问题。我们现有的所有协作工具，底层都假设使用者是"有顾虑的人"。权限设计、版本控制、冲突合并——这些机制能正常运转，是因为人类通常不会主动破坏文档。Agent 打破了这个假设。

我目前觉得比较靠谱的方向是：**Conductor + Draft Branch 的组合**。

核心思路是把"写"和"合并"分成两件事。每个 Agent 不直接操作主文档，而是在自己 fork 出来的草稿版本上自由工作——它可以随意写、随意改，不影响任何人。完成后，由一个专门的 Conductor Agent 来做 merge review：它通读所有草稿，决定哪些内容进主文档、哪些冲突如何取舍、整体结构和风格怎么保持一致。

这个模式的关键在于角色分离。生产内容的 Agent 不需要感知彼此的存在，它只管在自己的草稿里把事情做好；Conductor 不生产内容，它只负责协调，相当于一个编辑而不是作者。两个角色的目标不同，所以不会互相干扰。

和 git 的工作流几乎是同构的——每个 Agent 在自己的 branch 上 commit，Conductor 做 merge + code review，人类是最终的 merge 决策者。这套流程在工程协作里已经跑通了几十年，用在文档协作里没有理由行不通。

代价是实时感消失了。你看不到另一个 Agent 正在写什么，只能在它提交草稿之后才能看到。但对于真正需要多 Agent 协作的场景——策略文档、PRD、复杂分析报告——"实时"其实没那么重要，"最终产出的质量可控"才重要。

---

Proof 现在还很早期，多 Agent 协作的问题他们自己也还没解。但它做对了一件最重要的事：**把 Agent 当成第一用户来设计，而不是把 Agent 支持当成一个附加功能来做。**

这个差别，比听起来要大得多。

前者意味着你的产品摩擦点、权限模型、写入机制，每一个设计决策都要先问：Agent 用起来怎么样？后者只是在现有工具上加了一个"AI 助手"按钮。

左边的路正在被越来越多人走。但大多数人还在走右边那条——在 Notion 里加 AI 写作，在 Google Docs 里加 Gemini 助手。这不是在做 Agent-First，这是在给旧工具贴膏药。

哪条路更难走，哪条路更正确——我觉得这两件事指向同一个答案。
