Garry Tan 最近发了一个短视频讲"20x 公司"。视频不长，但信息密度很高——三个真实公司案例，三种不同的内部自动化路径。看完之后我没有那种"哇好厉害"的感觉，反而是一种"原来别人已经做到这一步了"的紧迫感。

因为他描述的东西，我正在小规模地实践。而这些公司，已经把它做到了组织级别。

---

## "20x"不是夸张，是一道算术题

"20x 公司"这个词是 Giga ML 的创始人提出的。Giga ML 做企业语音客服 agent，拿下 DoorDash 做客户的时候只有四五个工程师——竞争对手有 100 多个。所以他们说：我们是 20x 公司，因为我们能以 1/20 的人数打赢 20 倍大的对手。

Garry Tan 在此基础上把"20x"从一个形容词变成了一个方法论。他认为这不是个别天才公司的特例，而是一种新的创业范式：

> 不是只自动化一两个职能。而是自动化所有内部职能——代码、客服、市场、销售、招聘、QA。让每一个员工都比常规状态强一个量级。

他把这个概念跟 Parker Conrad 的"复合型创业公司（compound startup）"做了对比。Parker 的理论是——同时构建多个并行产品，能到达一种更强大的、更难被取代的 PMF。20x 公司是这个理念的内部版本：**不是构建多个产品，而是构建多层内部自动化。**

这个区分很重要。大多数人谈 AI 自动化，谈的是对外产品——"用 AI 给客户提供更好的服务"。但 20x 公司强调的是对内——用 AI 让自己的组织运转效率提高一个量级。产品只是结果，内部效率才是根因。

---

## 三条路径，不是选一个

视频里最有价值的部分是三个真实案例，分别代表三种不同的内部自动化路径。

**路径一：造一个 AI 队友。** Giga ML 构建了一个叫 Atlas 的内部 agent——能用浏览器、编辑策略、写代码，在产品内什么都能做。效果是每个工程师的工作范围翻了两到三倍。更夸张的是，他们整个公司只有一个人类全职客户服务人员，却在同时服务 DoorDash 和十多家财富 500 强的试点——每家每天的通话量在 50 万到 100 万通。

**路径二：造一个 AI 信息中台。** Legion Health 做 AI 原生的精神科诊疗网络。他们给护理运营团队做了一个内部界面——调取患者病史、排班、保险代码、消息记录——全部在一个地方。结果是营收增长了 4 倍，但没有多招一个人。整个运营团队：一个临床负责人、一个患者支持、一个账单人员。在传统医疗公司，这三个都是部门。

**路径三：为每个人造定制 agent。** Phase Shift 做应收账款自动化，12 人团队对抗 2006 年就成立的几百人公司。他们的做法很朴素：问每个员工"你每天花时间做什么"，让他们记录下来，然后为每个人构建专属的 AI agent。甚至连设计师都没招——前端设计全靠 Magic Patterns 加工程团队搞定。

Garry Tan 特意强调：**这三种路径不互斥。** 你可以同时做。做到的公司保持精简，并且在创造创纪录的增长率。

## 让我停下来想了很久的一句话

视频开头引用了 Anthropic 工程师的一段话：

> "我们人类线下碰头讨论基础架构和产品决策。但所有开发者同时管理着三到八个 Claude 实例，让它们实现功能、修 bug、研究方案。"

开发 Claude Code 的团队自己就是这样用 Claude Code 的。人类做架构决策，AI 做执行。一个开发者管三到八个 AI 实例。

这句话让我停下来想了一会儿，因为这恰好是我自己正在做的事的一个极端放大版。

我现在在 Claude Code 里装了 80 多个 skill，很多就是我自己的"内部自动化"——小红书数据分析、Obsidian 发公众号、播客转录、YouTube 转文档。每一个都是因为某件事我做了第三遍才封装的。一个人管多个 AI 执行实例，这个模式我已经在用了，只是规模小得多。

**Anthropic 的做法告诉我，这个模式的天花板远没有到。** 他们不是一个人管一个 AI 助手，是一个人管三到八个并发实例。这本质上是"人类做决策，AI 做所有执行"的组织范式。

## Phase Shift 的方法最值得学

三个案例里，我觉得 Phase Shift 的做法最有复制价值。

原因很简单：他们的方法不需要你先造出一个超级 agent 像 Atlas 那样，也不需要一个完整的信息中台。他们就是做了一件极其朴素的事——**问每个人"你每天在做什么"，记录下来，然后造 agent 来替代**。

这个逻辑我在自己做 skill 的过程中已经验证过了。我做 `/podcast-transcribe` 就是因为播客转录这件事我做了十几遍。做 `/xhs-scraper` 是因为抓小红书数据这件事每次都要手动来一遍。做 `/obsidian-to-wechat` 是因为发公众号的流程太烦了。

每一个 skill 的起点都是同一个问题：**这件事我是不是已经做了第三遍？**

Phase Shift 把这个逻辑制度化了——不是靠创始人自觉，而是要求每个员工记录自己的手动工作，然后系统性地用 agent 替代。这就是从"个人效率工具"到"组织级自动化"的跨越。

## 对我实际工作的启发

看完这个视频，我想到的第一件事不是"我该怎么创业"，而是"我现在的工作中，哪些环节还在重复手动做"。

在汽车行业做用户增长，有大量重复性工作是隐藏的。每周从各个渠道拉数据汇总留资量、对比不同城市的转化率、看 A/B 实验的结果然后写周报——这些我做了无数遍，但从来没有想过把它变成一个 agent。

Phase Shift 的做法给了我一个很具体的行动点：**花半天时间，把我上个月每天做的手动工作列出来。看看哪些能变成 skill 或 agent。**

不用等组织推动。一个产品经理能做的内部自动化，规模虽然小，但逻辑和 20x 公司完全一样——**让重复劳动消失，把时间留给只有人能做的判断。**
