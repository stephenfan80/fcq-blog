上个月我想给 Claude Code 加一个能力：自动把 Obsidian 笔记发布到微信公众号。我自己摸索了两天——研究微信 API、调试 token 刷新、处理富文本格式转换——才勉强跑通。

后来我发现有人已经把这套流程打包成了一个 skill。装上之后，三分钟就能用。

那一刻我意识到：这不就是 App Store 的逻辑吗？只不过卖的不是功能，是经验。

别人踩过的坑、试过的方案、打磨过的流程，变成一个文件夹，装上就能用。你不需要理解他怎么做的，你只需要知道你要什么。

这件事正在变成一个市场。

## 先说清楚：什么是 Agent Skills

一句话：**一个文件夹 + 一个 SKILL.md 文件 = 一个 Skill。**

2025 年 12 月 18 日，Anthropic 把 Agent Skills 作为开放标准发布。三个月后，**16 个以上的平台已经采纳了这个标准**：Claude Code、OpenAI Codex、Gemini CLI、Cursor、VS Code、GitHub Copilot、OpenClaw、Hermes Agent……

一个 skill 的结构极其简单：

```
xhs-content-audit/
├── SKILL.md          # 元信息 + 指令（唯一必需文件）
├── scripts/          # 可选：自动化脚本
├── references/       # 可选：参考文档
└── assets/           # 可选：模板、数据
```

SKILL.md 的核心就是两部分——YAML 头信息说明"我是谁、我能做什么"，Markdown 正文告诉 AI agent "遇到这类任务，按这个步骤来"。

如果 MCP 是"API 的 USB 接口"，那 Agent Skills 就是 **"经验的安装包"**。

截至目前，skills.sh 上已有 **44,000+ 个 skills**。MCP 生态从去年的 1,000 个 server 增长到了 10,000+。整个 agentic AI 市场在 2026 年达到 **91.4 亿美元**。

这不是一个概念，这是一个已经在运转的市场。

---

## 为什么这是一个新的增量市场

我有三个判断。

### 判断一：从"卖功能"到"卖经验"

过去二十年，软件行业的核心商业模式是卖功能。你写一个 App，实现某个功能，用户付费使用。**App 是功能的软件化。**

Agent Skills 改变了这个逻辑。**Skill 是经验的软件化。**

一个资深产品经理花了五年摸索出的 PRD 写作方法论——怎么定义问题、怎么拆用户故事、怎么做优先级排序——他可以把这套方法论打包成一个 skill。任何人装上之后，Claude Code 就会按照这套方法论来工作。

这和知识付费不一样。知识付费的交付物是"教你做"。Skill 的交付物是"替你做"。

你买一本《如何写好 PRD》的课，学完了还是你自己写。你装一个 PRD skill，AI 直接按照专家的方法论帮你写。

> **知识付费卖的是"知道"，Agent Skills 卖的是"会做"。交付门槛降了一个量级。**

这意味着一个新的供给出现了：任何在某个领域积累了深度经验的人，都可以成为 skill 开发者。不需要会写代码——一个 markdown 文件就是起点。

### 判断二：Write Once, Run Everywhere 终于实现了

做过移动开发的人都知道"一次编写、到处运行"这个口号喊了多少年，从来没真正实现过。iOS 和 Android 两套代码，微信小程序又是一套，Web 还得再来一次。

Agent Skills 做到了。

**你写一个 skill，它同时在 Claude Code、OpenAI Codex、Gemini CLI、Cursor、VS Code、GitHub Copilot、OpenClaw、Hermes Agent 上运行。** 不需要适配，不需要改代码，因为格式就是 markdown——所有平台都能读。

这意味着什么？skill 开发者的潜在市场规模 = 所有 AI 编码工具用户的总和。不是某一个平台的用户，而是全部。

做 iOS App 你只能触达 iOS 用户。做 Agent Skill 你能触达所有 agent 用户。这是移动互联网时代从未有过的分发效率。

### 判断三：中文市场几乎是空白

打开 skills.sh 或 SkillsMP，浏览一下已有的 skills。你会发现一个很明显的事实：**90% 以上是英文 skill，面向英文用户场景。**

中文世界的大量需求没有人在做：

- **小红书运营**：内容审核、选题规划、评论区互动策略
- **微信公众号**：从 Obsidian/Notion 一键发布、排版优化
- **飞书文档**：会议纪要自动生成、OKR 追踪
- **汽车之家/懂车帝**：经销商数据分析、竞品监控
- **抖音/快手**：短视频脚本生成、投流数据分析
- **电商**：1688 选品、淘宝详情页生成、客服话术

这些场景，海外开发者不会做、做不了、也没有动力做。

**中文 skill 市场有一个明确的窗口期。** 谁先做，谁定义品类。就像 2009 年 App Store 刚开放时，最早上架的 App 吃到了最大的自然流量。

## 怎么构建一个 Skill

五步。

**第一步：找到你反复在做的事。**

打开你最近一个月的工作记录。哪些任务你做了三次以上？哪些流程你每次都要从头来？重复就是机会。

我自己的例子：我每次分析小红书账号数据，都要手动抓取笔记、清洗数据、算互动率、出结论。这个流程重复了十几次之后，我把它写成了一个 skill。

**第二步：把操作步骤写成 SKILL.md。**

不需要写代码。用 markdown 写清楚：这个 skill 做什么、什么时候触发、按什么步骤执行、输出什么格式。就像你写一份 SOP 给新来的实习生——只不过这个实习生是 AI。

```yaml
---
name: xhs-content-audit
description: 分析小红书账号内容表现，生成互动率报告和选题建议。当用户提到小红书分析、内容审计时触发。
license: MIT
metadata:
  author: stephenfan
  version: "1.0"
---
```

**第三步：加上 scripts/ 放自动化脚本。**

如果你的 skill 需要执行代码（比如调用 API、处理数据），把脚本放在 `scripts/` 目录下。Python、Bash、JavaScript 都行。

**第四步：用 `skills-ref validate` 检查格式。**

官方提供了验证工具，确保你的 SKILL.md 格式正确、命名规范。

**第五步：在自己的 agent 上反复测试。**

装上跑几遍，看 AI 是不是按你预期的方式工作。边界情况怎么处理？出错了会怎样？这一步最关键——一个 skill 好不好用，不取决于你写了多少字，而取决于你测了多少遍。

**一个重要的认知：好的 skill 不是通用的，而是垂直的。**

"帮你写代码"太宽泛，Claude Code 本身就能做。"帮汽车经销商写小红书种草文，语气像真实车主，结尾带上指定门店地址和优惠信息"——这才是有价值的 skill。越垂直，越不可替代。

## 怎么发布和获客

### 发布渠道

现在主要有四个地方可以发布 skill：

| 平台 | 特点 |
|------|------|
| **skills.sh** | 最大的 skill 目录，44k+ skills，有安全扫描 |
| **SkillsMP** | 专注 Claude/Codex/ChatGPT 生态 |
| **ClawHub** | OpenClaw 社区生态 |
| **GitHub** | 直接开源，最大的自然流量 |

建议全部发布。一个 skill 发到四个平台，总共不到十分钟——因为本质上就是推一个 Git 仓库。

### 获客策略

**免费开源核心 skill，付费卖增值。**

把基础版 skill 开源。用户装了觉得有用，自然会想要更多。Pro 版本加上更精细的模板、更多的场景覆盖、定期的经验更新——这就是付费点。

**在目标用户聚集的地方做内容。**

你的用户在小红书？就去小红书写"我做了一个 XX skill，装上 Claude Code 就能自动 XX"。你的用户在即刻？就去即刻发。你的用户在 V2EX？就去 V2EX 发帖。

这不是"写一篇推广文"，而是"展示一个真实的成果"。最好的获客就是让人看到效果。

**做垂直领域合集。**

"新媒体运营 10 个必装 Agent Skills"、"产品经理的 5 个效率 Skills"——这种合集内容天然有传播力，因为它帮用户降低了筛选成本。

### 定价参考

三层定价模型，适用于不同阶段：

- **免费开源** = 获取用户 + 建立信任。适合起步期。
- **¥99-299 / skill** = 垂直专业场景的一次性购买。适合已验证需求。
- **¥1,500-3,000 / 月** = Skill + 持续运维 + 经验迭代。这不是卖 skill，是卖"数字员工服务"。

第三种定价，其实就是 AgentBox 模式——你卖的不是一个文件夹，而是一个持续在学习、在进化的 AI 员工。Skill 是载体，经验积累才是价值。

## 窗口期判断

**2026 年的 Agent Skills ≈ 2008 年的 App Store。**

但有一个关键区别：做一个 skill 的门槛比做一个 App 低两个数量级。

做 App 你需要：学编程、搞设计、注册开发者账号、提交审核、做多端适配。

做 Skill 你需要：写一个 markdown 文件。

这意味着什么？**供给端的爆发速度会远快于 App Store 时代。** 44,000 个 skills 在三个多月里就出现了，App Store 花了两年才达到类似的数量。

但供给快不代表质量高。大量 skill 是粗糙的、通用的、没人用的——就像 App Store 早期 90% 的 App 都是垃圾一样。

**真正的机会属于那些在特定领域有深度经验、愿意把经验打磨成高质量 skill 的人。**

如果你是做新媒体运营的，你比任何程序员都更懂"什么样的小红书标题有点击率"。如果你是做数据分析的，你比任何 AI 工程师都更懂"老板真正想看的报表长什么样"。如果你是做汽车销售的，你比任何人都更懂"客户说'再考虑考虑'的时候其实在想什么"。

这些经验，以前只能存在你的脑子里，或者在口耳相传中慢慢失真。现在，它们可以被安装。

> **你不需要会写代码。你需要的是，对某个领域足够了解，了解到可以教一个 AI 按你的方式做事。**

App Store 创造了移动互联网的十年繁荣。Agent Skills 正在创造的，可能是一个规模更大、准入门槛更低的新市场。

它刚刚打开。
